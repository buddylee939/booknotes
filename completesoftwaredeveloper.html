Chapter 3 - The technical skills you need to have

Learn One Programming Language

After or during: learn how to properly structure your code: Code Complete book

Object Oriented design: have a good understanding of what a class is, what the different types of inheritance are, and when to use them, understand terms like: polymorphism and encapsulation

Algorithms and Data structures: several algos that are commonly used for sorting lists of things programmatically. Data structures: arrays or vectors, linked lists, stacks, queues, trees, hashes, sets

JOIN TOPCODER

Read: Cracking the Coding interview, by Gayle Laakmann McDowell

A development platform and related technologies: Mac/Windows/Linux/Web

Framework of Stack: framework is simply a set of libraries that are used to develop code on a particular platform or on multiple platforms; A Stack is a set of technologies, usually including a framework, that are commonly used together to create a full application (MEAN stack, MongoDB, Express, Angular, Node)

Baisc Database Knowledge: relational databases and document databases; how they work, how to perform basic queries to get data, how to insert, update and delete data, how to join datasets together, and how to retrieve and store data programmatically from your code

Source Control: how to check in code, check out code, and hopefully merge changes from multiple sources;

Build and Deployment: after you check code into source control system, have some way to make sure that code actually works after you check it in; Build system is going to compile all the code and make sure there are no compliation errors; may run unit tests or user test, run code quality checks, and provide some reporting; A Deployment system will be responsible for deplyoing the code either to a production machine or perhaps to some kind of a test environment: You dont have to be an expert, but its pretty important that you understand at least the basics of how these systems work

Testing: terms like; whitebox testing, blackbox testing, unit testing (not really testing), boundary conditions, test automation, acceptance testing

Debugging: if you apply test-drive dev, youll spend less time in the debugging phase; learn to debug yourself or others code;

Methodologies: 'Scrum' search that methodology; be at least familiar with some of the basic ideas behind the most common software dev methodologies; Waterfall dev and Agile dev

Chapter 6 - Learning your first programming language

Start by looking at a working application

Look for an open source app on github

Find a few good resources or books and scan through them

Learn how to create hello world

create the most basic program

learn basic constructs and test them out with real problems:

here is a list of basic constructs most programming languages should contain

  1. ability to write output to the screen
  2. basic math capability
  3. storage of information as a variable
  4. organization of code into functions, methods, or modules
  5. invoke a function or method
  6. performance of boolean logic evaluations
  7. branch conditional statements(if/else)
  8. looping statements

that is the cord of programming in any language

know the difference between language features and libraries

programming today is more about knowing how to use libraries and frameworks than being an absolute expert in the language

review existing code and work through understanding each line: here is where people get stuck because they learnt how a function works, but don't know how to use it in the real world, so it is good to look at existing code, line by line, and making sure you understand exactly what each line and statement in the code is doing

build something, lots of somethings

here is a list of some simple project ideas to get you started:

  1. create a program that solves a mathematical problem by getting inputs from the user
  2. create a choose your own adventure type of program where the input from the user determines what happens next
  3. create a very simple text-based adventure game where the user can issue commands to pick up objects, move through rooms, etc
  4. create a program that is able to read input from a text file and write output to a different text file
  5. create a chatbot that talks to the user and pretends to be human or gives humorous responses

apply the programming language to a specific technology or platform

now you will be developing specific specialized skills with the programming language you are learning that will be useful for getting a job: start creating a few small applications using it

solve difficult algorithm problems with the language to master it

USE SITES LIKE TOPCODER to do competitions, new programming challenges every week at first he was horrible, couldn't solve the easiest problems, then by watching other people solve the problems and seeing their code, he got better and better

good sources for praticing algorithmic problems

Chapter 9 - teaching youself

Advantages

  1. cost is low or nothing
  2. self education is one of the most valuable skills you can learn
  3. you can learn at your own pace
  4. you can work around your schedule
  5. you can go deep on subjects you are interested in

Disadvantages

  1. you have to figure out what to do and what to learn
  2. no help with finding a job
  3. its easy to lose motivation
  4. social isolation
  5. likely to have gaps in knowledge: many self taught programmers lack some of the best practices and orthodoxy that college or boot camp graduates may have
  6. you can combat this by taking computer science material and working on shared projects, like open source

Strategies

  1. come up with a plan
  2. set a schedule: the cumulative effect is one of the most powerful effects for making progress in any area of development. making a little bit of steady progress each and every day or week
  3. network while you are learning: meetups and user groups
  4. START NOW
  5. find a mentor
  6. build at least one side project: give you a real problem to practice on and apply what you are learning; create a portfolio of real work you can use to show at an interview; create some possible side income or a new business; create some useful toos for youself while you are learning to code; build confidence in your abilities; provide and outlet for fun
  7. subscribe to a site like 'pluralsight'
