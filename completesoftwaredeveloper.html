  1. contact all the people in my networks to let them know what value I can provide and what I am looking for

Chapter 3 - The technical skills you need to have

Learn One Programming Language

After or during: learn how to properly structure your code: Code Complete book

Object Oriented design: have a good understanding of what a class is, what the different types of inheritance are, and when to use them, understand terms like: polymorphism and encapsulation

Algorithms and Data structures: several algos that are commonly used for sorting lists of things programmatically. Data structures: arrays or vectors, linked lists, stacks, queues, trees, hashes, sets

JOIN TOPCODER

Read: Cracking the Coding interview, by Gayle Laakmann McDowell

A development platform and related technologies: Mac/Windows/Linux/Web

Framework of Stack: framework is simply a set of libraries that are used to develop code on a particular platform or on multiple platforms; A Stack is a set of technologies, usually including a framework, that are commonly used together to create a full application (MEAN stack, MongoDB, Express, Angular, Node)

Baisc Database Knowledge: relational databases and document databases; how they work, how to perform basic queries to get data, how to insert, update and delete data, how to join datasets together, and how to retrieve and store data programmatically from your code

Source Control: how to check in code, check out code, and hopefully merge changes from multiple sources;

Build and Deployment: after you check code into source control system, have some way to make sure that code actually works after you check it in; Build system is going to compile all the code and make sure there are no compliation errors; may run unit tests or user test, run code quality checks, and provide some reporting; A Deployment system will be responsible for deplyoing the code either to a production machine or perhaps to some kind of a test environment: You dont have to be an expert, but its pretty important that you understand at least the basics of how these systems work

Testing: terms like; whitebox testing, blackbox testing, unit testing (not really testing), boundary conditions, test automation, acceptance testing

Debugging: if you apply test-drive dev, youll spend less time in the debugging phase; learn to debug yourself or others code;

Methodologies: 'Scrum' search that methodology; be at least familiar with some of the basic ideas behind the most common software dev methodologies; Waterfall dev and Agile dev

Chapter 4 - How to develop technical skills

Learning how to learn quickly

The basic process: first get a good idea of what you are learning and what the scope of it is

get a big picture, and narrow down to a small enough scope, so you can tackle it and wrap your head around it

then you need a goal and what metric you will use to know that youve learned it

gather some resources for learning

gather multiple resources, books, blogs, podcasts, magazines, video courses, tuts, expert opinions

create a plan for learning based on those resources

use the table of contents to help figure out what order you should try to learn things

then teach it to someone else

Learn by Doing:

git side note:merging a file into the wrong branch, or checking out the wrong version of the source code, used a version history to figure out where a bug got introduced

How to learn by doing: first start by figuring out what it is going to help you do;

An example of learning by doing: relational databases and how to use them; instead of just learning, think of what you want to do with, say create a movie database that I own; insert to it, delete, update etc then search and learn how to achieve that!

How I Teach technical skills

3 things I try to teach someone when I am teaching a technical skill: the big picture, what can you do with the technology; how to get started; the 20% you need to know to be the most effective

The Big Picture: history of the language and what it is used for; all the constructs of the language and language features, especially unique ones; various libraries that are standard part of the language;

How to get started: download what ever is needed, get it installed, create their first project, and compile their code

You need to know 20% to be the most effective: in the example of database: learn how to write select statements, join tables and other commong operations

Read what experts are writing: spend 30 minutes each day reading various blogs related to subjects I was learning about;
